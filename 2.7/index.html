Sidlayout

webbsideslayout

När man pratar om webbsideslayout så pratar man ofta om själva uppbyggnaden av en webbsida. Vilka grundprinciper som ska gälla och vilka element man skan använda sig av. Tre kända principer som finns är Fixerad, Flytande och Elastisk. Kallat på engelska för Fixed, fluid och Elastic. Dessa tre går ut på att bestämma hur de olika elementen på en websida ska bete sig på olika skärmar och datorer. Exempel på hur sidor med de olika priciperna kan se ut hittar ni i listan nedan.
Det vanligaste är att man i en webbsideslayout visar inehållet på en sida i olika kollumner. Man placerar sedan ut dessas kollumner på sidan enligt den design som man vill uppnå.

  • Fixerad design
  • Flytande design
  • Elastisk design

Fixerad design

En fixerad design betyder att man skapar ett element som fungerar som ett omslag för de andra collumnerna, detta omslag har sedan ett fast värde för sin bredd. De kollumner som man sedan lägger innuti detta omslag kan har bredd baserat på procent eller ett fast värde(så som pixlar), men det viktiga är att omslaget inte ändrar storlek oavsett vilken upplösning användarens skärm har.

Flytande design

En flytande design har den större delen av element en procentuell bredd och den kommer därför att anpassa sig beroende på vilken typ av upplösning användaren har på sin skärm. Vissa enheter på siten kan sättas med en fast bredd men majoriteten och den allmänna principen ska vara att använda sig av procentuella värden när det kommer till bredd.

Elastisk design

En elastisk design är lite av en blandning mellan Fixerad och Flytande design. Denna typ av design är baserad på att man använder sig av em för att bestämma vilken storlek det ska vara på element som finns på sidan. Skillnaden mellan en pixel och em är att en pixel är ett helt fast värde som alltid kommer vara den samma på användarens skärm, men en em representerar fontens storlek och dess storlek blir därför alltid relativ till vilken textstorlek användaren har på sin skärm.

CSS vs tabeller

En gammal metod för att skapa en sidlayout var att använda sig av HTML elementet <table> och på så sätt skapa olika kollumner som man sedan kunda placera ut och ändra storlekar på. Detta anses dock numera som ett dåligt sätt att designa en sida på. En av anledningarna till detta är att <table>-taggen aldrig var designad för att skapa en layout, syftet för denna tag är att publicera tabellstrukturerad data. Detta gör att koden blir väldigt strulig och svår att läsa, det blir även extremt svårt att göra förändringar i designen vid ett senare skeden.

Positions egenskaper

position

Egenskapen position specificerar den typ av positionering som används på ett element. Den kan vara statisk, relativ, absolut eller fixerad.

Top

Egenskapen top används för att sätta ett avstånd från elementets övre kant till den övre kanten på sin närmsta positionerade förälder. Ett positionerat element innebär ett element som inte har värdet static satt på egenskapen position.
Värt att nämna är även att ifall ett absolut positionerat element inte har någon positionerad förälder så kommer den att använda sig av dokumentets <body>-tagg för denna uträkning.

Left

Egenskapen left används för att sätta ett avstånd från elementets vänstra kant till den vänstra kanten på sin närmsta positionerade förälder. Ett positionerat element innebär ett element som inte har värdet static satt på egenskapen position.
Värt att nämna är även att ifall ett absolut positionerat element inte har någon positionerad förälder så kommer den att använda sig av dokumentets <body>-tagg för denna uträkning.

Bottom

Egenskapen bottom används för att sätta ett avstånd från elementet nedre kant till den undre kanten på sin närmsta positionerade förälder. Ett positionerat element innebär ett element som inte har värdet static satt på egenskapen position.
Värt att nämna är även att ifall ett absolut positionerat element inte har någon positionerad förälder så kommer den att använda sig av dokumentets <body>-tagg för denna uträkning.

Right

Egenskapen right används för att sätta ett avstånd från elementet högra kant till den höra kanten på sin närmsta positionerade förälder. Ett positionerat element innebär ett element som inte har värdet static satt på egenskapen position.
Värt att nämna är även att ifall ett absolut positionerat element inte har någon positionerad förälder så kommer den att använda sig av dokumentets <body>-tagg för denna uträkning.

Z-index

Egenskapen z-index används för att sätta prioritering på vilket element som ska ligga ovanpå andra element. Ett element med högre z-index ligger alltid ovanpå ett element med ett lägre z-index.
Z-index fungerar dock endast på positionerade element, alltså element som har något av nedan tre värden på egenskapen position.

  • absolut
  • relative
  • fixed

Float

Egenskapen float specificerar ifall ett element ska vara "flytande" eller ej. Denna egenskap ignoreras dock ifall ett element har en absolut positionering.

clear

Egenskapen clear bestämmer om samt på vilken sida ett "flytande" element inte är tillåtet. Sitter värdet "left" så får ingenting flyta till vänster om elementet.

Storleks egenskaper

Width

Egenskapen width sätter bredden på ett element, denna egenskap räknar inte med padding, border eller margin. Den sätter alltså bredden innanför padding och margin.

Height

Egenskapen height sätter höjden på ett element, denna egenskap räknar inte med padding, border eller margin. Den sätter alltså höjden innanför padding och margin.

max-width

Egenskapen max-width sätter den maximala bredden på ett element. Detta förhindrar att värdet på width egenskapen blir större än max-width.
Notera även att värdet på max-width alltid väger tyngre än värdet på width.

max-height

Egenskapen max-height sätter den maximala höjden på ett element. Detta förhindrar att värdet på height egenskapen blir större än max-height.
Notera även att värdet på max-height alltid väger tyngre än värdet på height.

min-width

Egenskapen min-width sätter den minsta bredden på ett element. Detta förhindrar att värdet på width egenskapen blir lägre än min-width.
Notera även att värdet på min-width alltid väger tyngre än värdet på width.

min-height

Egenskapen min-height sätter den minsta höjden på ett element. Detta förhindrar att värdet på height egenskapen blir lägre än min-height.
Notera även att värdet på min-height alltid väger tyngre än värdet på height.

Vertical-align

Egenskapen vertical-align sätter den vertikala linjeringen för ett element.

CSS kollumner

Förutom de separata kollumner som man kan bygga upp med hjälp av css och div element så finns det även en inbyggd för funktion för kollumner i CSS som direkt delar upp ett element i flera kollumner.
Det finns ganska många egenskaper för detta och det går till exempel att läsa mer om dem på W3 schools. men i denna artikel tänkte jag endast titta närmare på två av dom, column-count och column-width.

Column-count

Column-count används för att bestämma hur många kollumner som ska finnas i enheten.

Column-width

Column-width används för att välja bredden på varje kollumn som finns i elementet.

Nedan har jag skapat ett exempel där jag har satt Column-count till 3 och column-width till 50px. Notera att när man använder sig av denna typ av kollumner så flödar texten över till nästa kollumne när den första är full.

Exempel

div {
column-count: 3;
column-width: 50px;
}
